# Tools Reference

Forge provides a comprehensive set of built-in tools that enable AI agents to interact with your system, manage files, process data, and communicate with users. This reference documents all available tools, their parameters, and usage examples.

## User Interface Tools

### tool_forge_display_show_user

Displays formatted markdown content to the user in the terminal.

```javascript
tool_forge_display_show_user({
  content: "# Hello World\n\nThis is **formatted** markdown content."
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `content` | string | Yes | Markdown content to display to the user |

#### Features

- Rich markdown formatting in terminal output
- Support for headers, lists, tables, code blocks, and text emphasis
- Properly rendered using the termimad library
- Improves readability of complex information

#### Example: Creating a Structured Output

```javascript
tool_forge_display_show_user({
  content: `# Analysis Results

## Performance Issues
- **High CPU Usage**: 94% average over 5 minutes
- **Memory Leak Detected**: Growing by ~5MB per minute

## Recommended Actions
1. Check process ID 3421 for excessive resource consumption
2. Review recent code changes in authentication module

| Metric | Current | Threshold | Status |
|--------|---------|-----------|--------|
| CPU    | 94%     | 80%       | ❌     |
| Memory | 2.3GB   | 2GB       | ❌     |
| Disk   | 68%     | 90%       | ✅     |

\`\`\`bash
# Run this command to restart the affected service
sudo systemctl restart app-server.service
\`\`\``
})
```

#### When to Use

Use this tool when you need to present:
- Structured information with clear hierarchy
- Tabular data
- Code snippets with syntax highlighting
- Step-by-step instructions
- Visual emphasis for important points

## File System Tools

### tool_forge_fs_read

Reads file contents at a specified path.

```javascript
tool_forge_fs_read({
  path: "/absolute/path/to/file.txt"
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Absolute path to the file |

### tool_forge_fs_create

Creates or overwrites a file with specified content.

```javascript
tool_forge_fs_create({
  path: "/absolute/path/to/file.txt",
  content: "File content goes here",
  overwrite: true
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Absolute path to the file |
| `content` | string | Yes | Content to write to the file |
| `overwrite` | boolean | No | Whether to overwrite if file exists (default: false) |

### tool_forge_fs_patch

Modifies files with targeted text operations on matched patterns.

```javascript
tool_forge_fs_patch({
  path: "/absolute/path/to/file.txt",
  patches: [
    {
      search: "old text",
      operation: "replace",
      content: "new text"
    }
  ]
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Absolute path to the file |
| `patches` | array | Yes | List of patch operations to apply |

#### Patch Operations

| Operation | Description |
|-----------|-------------|
| `prepend` | Add content before the matched text |
| `append` | Add content after the matched text |
| `replace` | Replace the matched text with new content |
| `swap` | Exchange the matched text with another text |

### tool_forge_fs_remove

Removes a file at a specified path.

```javascript
tool_forge_fs_remove({
  path: "/absolute/path/to/file.txt"
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Absolute path to the file |

### tool_forge_fs_search

Searches for text patterns across files using regex.

```javascript
tool_forge_fs_search({
  path: "/absolute/path/to/directory",
  regex: "function\\s+\\w+\\s*\\(",
  file_pattern: "*.js"
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Directory to search in (recursively) |
| `regex` | string | Yes | Regular expression pattern to search for |
| `file_pattern` | string | No | Glob pattern to filter files (e.g., "*.js") |

## Network Tools

### tool_forge_net_fetch

Retrieves content from URLs as markdown or raw text.

```javascript
tool_forge_net_fetch({
  url: "https://example.com/api/data",
  raw: false,
  max_length: 20000,
  start_index: 0
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `url` | string | Yes | URL to fetch content from |
| `raw` | boolean | No | Whether to return raw content instead of markdown (default: false) |
| `max_length` | number | No | Maximum characters to return (default: 40000) |
| `start_index` | number | No | Starting character index for pagination (default: 0) |

## Process Tools

### tool_forge_process_shell

Executes shell commands with safety measures.

```javascript
tool_forge_process_shell({
  command: "ls -la",
  cwd: "/absolute/path/to/directory"
})
```

#### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `command` | string | Yes | Shell command to execute |
| `cwd` | string | Yes | Working directory for command execution |

## Best Practices

### User Interface

1. **Use Structured Formatting**: Organize information with clear headings, lists, and emphasis
2. **Code Block Usage**: Always use code blocks for commands, code snippets, and configuration examples
3. **Table Organization**: Present comparative or structured data in tables for clarity

### File Operations

1. **Path Safety**: Always use absolute paths for file operations
2. **Error Handling**: Check for success or failure responses from file tools
3. **Minimize Writes**: Use `tool_forge_fs_patch` for small changes instead of rewriting entire files
4. **Content Verification**: Use `tool_forge_fs_read` to verify file content before modifying

### Shell Commands

1. **Command Safety**: Avoid potentially destructive commands
2. **Working Directory**: Always specify the appropriate working directory
3. **Command Results**: Always check command outputs and exit codes for errors

## Tool Selection Guide

| When you need to... | Use this tool |
|---------------------|--------------|
| Present formatted information to the user | `tool_forge_display_show_user` |
| Read file content | `tool_forge_fs_read` |
| Create or update a file | `tool_forge_fs_create` |
| Make small targeted changes to a file | `tool_forge_fs_patch` |
| Delete a file | `tool_forge_fs_remove` |
| Find text patterns across multiple files | `tool_forge_fs_search` |
| Retrieve data from a URL | `tool_forge_net_fetch` |
| Execute shell commands | `tool_forge_process_shell` |
